# How to report bugs

Found a bug in a White Ravens service? Great that you want to report it! By doing so, you help us improve our projects. Bugs can be reported on GitHub or Discord.

!!! tip "White Ravens Repositories"
    [GitHub White Ravens](https://github.com/orgs/whiteravens20/repositories)

---

## Quick path: report on Discord

Don't have a GitHub account or not sure where to report? Simply write on our [Discord server](https://discord.gg/5JMk8Z4) — we'll help.

---

## Reporting on GitHub — step by step

### Step 1: Create an account

If you don't have a GitHub account, go to [github.com](https://github.com/) → **Sign up** and create a free account.

### Step 2: Find the right repository

!!! info "Where to report?"
    Problems with White Ravens services (e.g., site not working, configuration error) should be reported in the [whiteravens20/wrservices.link](https://github.com/whiteravens20/wrservices.link) repository.

    If you're not sure where to report — report there, and we'll redirect it to the right place.

### Step 3: Check if someone already reported it

1. Go to the **Issues** tab in the repository.
2. Type keywords in the search bar.
3. If you find a similar report — you can add a comment with additional information.

### Step 4: Create a new issue

1. Click **New Issue**.
2. Choose a template:

    | Template | When to use |
    |---|---|
    | **Bug Report** | Something isn't working properly |
    | **Documentation Issue** | Error in documentation |
    | **New Feature / Service Request** | Suggestion for a new feature or service |
    | **Blank issue** | Other — none of the above fits |

3. Fill in the form — describe the problem as you see it.
4. Click **Submit new issue**.

!!! tip "Language"
    We prefer English on GitHub, but it's not a requirement — you can also write in Polish.

### Step 5: Track the issue

After submitting, you'll receive notifications about comments and status changes. If developers ask questions — respond to help resolve the problem.

---

## Tips — how to write a good report

- **Describe what happened** — what you were doing, what should have happened, and what happened instead
- **Add a screenshot** — a picture is worth a thousand words
- **Specify browser and system** — e.g., "Chrome on Windows 11" or "Firefox on Android"
- **Be specific** — the more details, the faster the problem will be fixed

---

## Frequently asked questions

??? question "Do I need a GitHub account?"
    Not necessarily. You can also report a problem on our Discord — someone from the team will create an issue for you.

??? question "I don't know which repository to report in."
    Report in [whiteravens20/wrservices.link](https://github.com/whiteravens20/wrservices.link) — we'll redirect it to the right place.

??? question "Can I suggest a new feature?"
    Yes! Choose the **New Feature / Service Request** template when creating a new issue.
